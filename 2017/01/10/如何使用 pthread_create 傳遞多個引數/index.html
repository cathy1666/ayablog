如何使用 pthread_create 傳遞多個引數

Posted on 2017-01-10

函數定義

#include <pthread.h>
int pthread_create( pthread_t *thread, 
                    const pthread_attr_t *attr, 
                    void *(*start_routine) (void *),
                    void *arg);
  • pthread_t *thread 為pthread的指標，在使用thread之前必須要先宣告一個pthread_t的變數。
  • const pthread_attr_t *attr 為一個structure指標，內含創建此thread的屬性，預設是NULL，如果沒有其他特殊的需求直接填入NULL即可。
  • void *(*start_routine) (void *) 為function pointer，即所創建的thread要執行的handler function。
  • void *arg 為Function pointer所指function要帶的參數,需為指標型態。

RETURN VALUE: 如果執行成功則回傳0，如果執行失敗則回傳錯誤代碼。

EX:

void *print_data(void *data);
...
pthread_t t1;
int arg1 = 5;
pthread_create( &t1, NULL , print_data , (void *)&arg1);

pthread_create( &t1, NULL , (void *)print_data , (void *)&arg1);

傳遞一個引數

void *print_data(void *data)
{
    //printf("data = %d\n", *data);//compile不會過

    int tt = *(int*)data;

    printf("print_data START\n");
    printf("data = %d\n", tt);
    printf("print_data END\n");

    pthread_exit(NULL);
}


int main()
{
    int arg1 = 88;
    pthread_t t1;

    if (pthread_create(&t1, NULL, print_data, (void *)&arg1) != 0)
    {
        printf("create thread fail: %s\n", strerror(errno));
    }
    sleep(5); //避免thread還沒執行完main就結束
    return 0;
}

(void *)指標型態的參數傳進function後，需要先指定是甚麼型態的指標才能使用，直接使用指標指向的值，compiler無法判斷要讀多長的記憶體空間，所以連compile都不會過:

Compile Error:

warning: dereferencing 'void *' pointer [enabled by default]
error: invalid use of void expression

傳遞多個引數

pthread_create的函數定義裡，只有一個欄位可以放傳入handler function的引數，所以要傳遞多個引數時，可以把需要的引數包成一個structure，再將此structure的pointer作為引數傳入。

struct testdata
{
    int num;
    char name[10];
};

void *print_data(void *data)
{
    struct testdata* tt = (struct testdata*)data;

    printf("print_data START\n");
    printf("num = %d\n", tt->num);
    printf("name = %s\n", tt->name);
    printf("print_data END\n");

    pthread_exit(NULL);
}

int main()
{
    struct testdata data1;
    pthread_t t1;

    data1.num = 100;
    strcpy(data1.name, "Hello");

    if (pthread_create( &t1, NULL , print_data , (void *)&data1) != 0)
    {
        printf("create thread fail: %s\n", strerror(errno));
    }
    sleep(5); //避免thread還沒執行完main就結束
    return 0;
}

pass by reference引起的segmentation fault

因為受限於pthread_create本身的定義，pthread_create所有的參數都是使用指標，也就是用pass by reference的方式傳遞引數，所以就要特別注意reference的位置是否為NULL，會不會造成segmentation fault。
以下例子就是我自己寫multi-thread程式的血淚經驗中得來的:

struct testdata
{
    int num;
    char name[10];
};

void *print_data(void *data)
{
    struct testdata* tt = (struct testdata*)data;

    printf("print_data START\n");
    printf("num = %d\n", tt->num);
    printf("name = %s\n", tt->name);
    printf("print_data END\n");

    pthread_exit(NULL);
}    

void func()
{
    struct testdata data1;
    pthread_t t1;

    data1.num = 100;
    strcpy(data1.name, "Hello");

    if (pthread_create( &t1, NULL , print_data , (void *)&data1) != 0)
    {
        printf("create thread fail: %s\n", strerror(errno));
    }
}

int main()
{
    func();

    sleep(5); //避免thread還沒執行完main就結束
    return 0;
}

OUTPUT

print_data START
num = -6947704
name = ��y���Z�u
print_data END

以上程式有很大的機率會在執行時發生segmentation fault，原因是傳入的引數data1是定義在func()內，func()一結束區域變數data1就會被系統清除，而此時thread進行到
printf("num = %d\n", tt->num);
這行時，就會讀錯記憶體區塊，造成segmentation fault。

解決方法

雖然所有pthread_create的動作都在main進行是最安全的作法，但程式越來越大，為了使程式結構化寫成function還是有必要的。所以我目前的解法是在thread handler function內一開始就把傳入指標指向的值copy一份，另外在pthread_create後也稍微delay確保copy引數值的動作完成才結束func()。

struct testdata
{
    int num;
    char name[10];
};

void *print_data(void *data)
{
    struct testdata tt = *(struct testdata*)data; // 直接copy值，而不是用reference的方法使用

    printf("print_data START\n");
    printf("num = %d\n", tt.num);
    printf("name = %s\n", tt.name);
    printf("print_data END\n");

    pthread_exit(NULL);
}

void func()
{
    struct testdata data1;
    pthread_t t1;

    data1.num = 100;
    strcpy(data1.name, "Hello");

    if (pthread_create(&t1, NULL, print_data, (void *)&data1) != 0)
    {
        printf("create thread fail: %s\n", strerror(errno));
    }
    usleep(500000);//確保copy引數值的動作完成才結束func()
}

int main()
{
    func();

    sleep(5); //避免thread還沒執行完main就結束
    return 0;
}

OUTPUT

print_data START
num = 100
name = Hello
print_data END

當然，這不是最好的做法，畢竟做了delay，如果有發現更好的做法再來更新吧!
