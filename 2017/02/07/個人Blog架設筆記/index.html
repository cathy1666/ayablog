(I)
原因是hexo默认是以项目根目录下的.deploy_git作为git仓库，所以我们得把这个仓库与线上的xxx.github.io仓库绑定，
如果绑定过其它目录下的仓库，可以直接把隐藏目录.git复制到.deploy_git目录下。
再次运行hexo d
(II)
$ rm -rf .deploy
$ hexo deploy

$ hexo init //此不一定要跟github上的Repo一樣
$ cd
$ npm install

3.
修改內的_config.yml

url: http://cathy1666.github.io/ayablog

root: /ayablog/

//如果是用gh-page的方式deploy,_config.yml的root要改為你repository的名字 root: /your_repository/ 不然連到https://.github.io//版面會跑掉只剩文字

deploy:
type: git
repository: git@github.com:/.git
branch: gh-pages

4.
$ hexo g //==hexo deploy #?建???面
$ hexo d //==hexo deploy #?布：清空.deploy_git/里文件，?public/复制新生成的文件

若發生 ERROR Deployer not found: git，請執行以下指令試試
npm install hexo-deployer-git –save
再
$ hexo g //==hexo deploy
$ hexo d //==hexo deploy

先hexo g再?行hexo d布署，也可使直接用hexo d -g

5.
$ hexo server //==hexo s, 發佈前可以使用hexo的server功能來預覽網站
http://localhost:4000/ 可先在本地端預覽

https://.github.io//

Q:
部署到github上時每次都要輸入入密?。如何免密碼部署呢？

A:
github?例：
使用https需要密碼，如下：
deploy:
type: git
repo: https://github.com/yourname/yo…
branch: master
使用ssh不需要密碼，如下：
deploy:
type: git
repo: git@github.com:yourname/yourname.github.io.git
branch: master
前提是配置好了github的ssh，也就是手動部署可以無需密碼了

github为例：

type: git
repo: https://github.com/yourname/yo…
branch: master
使用ssh不需要密码，如下：
deploy:
type: git
repo: git@github.com:yourname/yourname.github.io.git
branch: master
前提是配置好了github的ssh，也就是手动部署可以无需密码了
