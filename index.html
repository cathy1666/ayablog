不定長度引數函數的寫法

Posted on 2017-01-10

需求

把開 特定檔案、寫入帶有當下時間的log、關檔案 整個流程寫成一個function，且此function要有像printf的介面，可以自動偵測印出含有任意數量的變數的string。

Sample Code

int printLog(char* hex, int len, char* char_p, ...)
{
    int i;
    FILE *fp;
    unsigned char* hex_p = (unsigned char*)hex;

    fp = fopen(LOGFILE, "a");
    if (NULL == fp)
    {
        printf("open %s fail\n", LOGFILE);
        return -1;
    }    

    char buffer[256];
    va_list list_p;
    va_start(list_p, char_p);
    vsprintf(buffer, char_p, list_p);
    va_end(list_p);

    //print time stamp
    fprintf(fp, "%.24s   ", getTime()); 

    //print comment with variable-length argument 
    fprintf(fp, "%s", buffer);

    //print hex string    
    fprintf(fp, "[ ");
    for (i = 0; i < len; i++)
    {
        fprintf(fp, "%02x ", *hex_p++);
    }
    fprintf(fp, "]\n");

    fclose(fp);
    return 0;    
}

這段code是reference內sample code變化應用，想知道細節就參考reference吧，裡面寫得很詳細!

如何使用 pthread_create 傳遞多個引數

Posted on 2017-01-10

函數定義

#include <pthread.h>
int pthread_create( pthread_t *thread, 
                    const pthread_attr_t *attr, 
                    void *(*start_routine) (void *),
                    void *arg);
  • pthread_t *thread 為pthread的指標，在使用thread之前必須要先宣告一個pthread_t的變數。
  • const pthread_attr_t *attr 為一個structure指標，內含創建此thread的屬性，預設是NULL，如果沒有其他特殊的需求直接填入NULL即可。
  • void *(*start_routine) (void *) 為function pointer，即所創建的thread要執行的handler function。
  • void *arg 為Function pointer所指function要帶的參數,需為指標型態。

RETURN VALUE: 如果執行成功則回傳0，如果執行失敗則回傳錯誤代碼。

EX:

void *print_data(void *data);
...
pthread_t t1;
int arg1 = 5;
pthread_create( &t1, NULL , print_data , (void *)&arg1);

pthread_create( &t1, NULL , (void *)print_data , (void *)&arg1);

傳遞一個引數

void *print_data(void *data)
{
    //printf("data = %d\n", *data);//compile不會過

    int tt = *(int*)data;

    printf("print_data START\n");
    printf("data = %d\n", tt);
    printf("print_data END\n");

    pthread_exit(NULL);
}


int main()
{
    int arg1 = 88;
    pthread_t t1;

    if (pthread_create(&t1, NULL, print_data, (void *)&arg1) != 0)
    {
        printf("create thread fail: %s\n", strerror(errno));
    }
    sleep(5); //避免thread還沒執行完main就結束
    return 0;
}

(void *)指標型態的參數傳進function後，需要先指定是甚麼型態的指標才能使用，直接使用指標指向的值，compiler無法判斷要讀多長的記憶體空間，所以連compile都不會過:

Compile Error:

warning: dereferencing 'void *' pointer [enabled by default]
error: invalid use of void expression

傳遞多個引數

pthread_create的函數定義裡，只有一個欄位可以放傳入handler function的引數，所以要傳遞多個引數時，可以把需要的引數包成一個structure，再將此structure的pointer作為引數傳入。

struct testdata
{
    int num;
    char name[10];
};

void *print_data(void *data)
{
    struct testdata* tt = (struct testdata*)data;

    printf("print_data START\n");
    printf("num = %d\n", tt->num);
    printf("name = %s\n", tt->name);
    printf("print_data END\n");

    pthread_exit(NULL);
}

int main()
{
    struct testdata data1;
    pthread_t t1;

    data1.num = 100;
    strcpy(data1.name, "Hello");

    if (pthread_create( &t1, NULL , print_data , (void *)&data1) != 0)
    {
        printf("create thread fail: %s\n", strerror(errno));
    }
    sleep(5); //避免thread還沒執行完main就結束
    return 0;
}

pass by reference引起的segmentation fault

因為受限於pthread_create本身的定義，pthread_create所有的參數都是使用指標，也就是用pass by reference的方式傳遞引數，所以就要特別注意reference的位置是否為NULL，會不會造成segmentation fault。
以下例子就是我自己寫multi-thread程式的血淚經驗中得來的:

struct testdata
{
    int num;
    char name[10];
};

void *print_data(void *data)
{
    struct testdata* tt = (struct testdata*)data;

    printf("print_data START\n");
    printf("num = %d\n", tt->num);
    printf("name = %s\n", tt->name);
    printf("print_data END\n");

    pthread_exit(NULL);
}    

void func()
{
    struct testdata data1;
    pthread_t t1;

    data1.num = 100;
    strcpy(data1.name, "Hello");

    if (pthread_create( &t1, NULL , print_data , (void *)&data1) != 0)
    {
        printf("create thread fail: %s\n", strerror(errno));
    }
}

int main()
{
    func();

    sleep(5); //避免thread還沒執行完main就結束
    return 0;
}

OUTPUT

print_data START
num = -6947704
name = ��y���Z�u
print_data END

以上程式有很大的機率會在執行時發生segmentation fault，原因是傳入的引數data1是定義在func()內，func()一結束區域變數data1就會被系統清除，而此時thread進行到
printf("num = %d\n", tt->num);
這行時，就會讀錯記憶體區塊，造成segmentation fault。

解決方法

雖然所有pthread_create的動作都在main進行是最安全的作法，但程式越來越大，為了使程式結構化寫成function還是有必要的。所以我目前的解法是在thread handler function內一開始就把傳入指標指向的值copy一份，另外在pthread_create後也稍微delay確保copy引數值的動作完成才結束func()。

struct testdata
{
    int num;
    char name[10];
};

void *print_data(void *data)
{
    struct testdata tt = *(struct testdata*)data; // 直接copy值，而不是用reference的方法使用

    printf("print_data START\n");
    printf("num = %d\n", tt.num);
    printf("name = %s\n", tt.name);
    printf("print_data END\n");

    pthread_exit(NULL);
}

void func()
{
    struct testdata data1;
    pthread_t t1;

    data1.num = 100;
    strcpy(data1.name, "Hello");

    if (pthread_create(&t1, NULL, print_data, (void *)&data1) != 0)
    {
        printf("create thread fail: %s\n", strerror(errno));
    }
    usleep(500000);//確保copy引數值的動作完成才結束func()
}

int main()
{
    func();

    sleep(5); //避免thread還沒執行完main就結束
    return 0;
}

OUTPUT

print_data START
num = 100
name = Hello
print_data END

當然，這不是最好的做法，畢竟做了delay，如果有發現更好的做法再來更新吧!

如何外部終止特定執行緒

Posted on 2017-01-05

需求

主程式會不定時創建 5 個執行緒執行工作A~E, 當主程式需執行A工作時，需先判斷上一輪的A是否結束，若還沒結束就需先強制終止掉正在進行A的執行緒後，再創建一個新的執行緒執行這次的工作A。

問題

使用pthread_cancel()來外部終止執行緒是以pthread_id來識別，但pthread_id是類似file descriptor的識別碼，只要空了就可以重複使用，所以有可能上一輪的A已經結束，而此pthread_id被後來創建的執行緒C拿去用。所以用pthread_id在多執行緒的環境中無法識別執行特定工作的執行緒是否還沒結束，也就是說，用pthread_cancel()可能會終止掉非目標的執行緒，造成程式錯誤。

解法

不使用pthread_id來識別執行緒，而是自己定義一個陣列記錄每個執行緒目前的狀態。但此陣列可能同時被多個執行緒存取，造成race condition, 所以需再加上mutex做保護。

Sameple code

#include <stdio.h>
#include <unistd.h>
#include <pthread.h>


#define DEV_MAX 5
volatile int SendTrigThreadStop[DEV_MAX] = { 0 };
pthread_mutex_t TrigStopMutex[DEV_MAX];
pthread_t thread_SendTrigMsg[DEV_MAX];


    void *func(void *param)
    {    
        //pthread_detach(pthread_self());
        //在pthread_create後，在thread function中pthread_detach自己,資源在thread function退出時或調用pthread_exit()時會自動被釋放
        //加了這行就不能被join了


        int num = (int)param;
        long long int count = 0;
        printf("thread %d begin...\n", num);


        while (getTrigThreadStat(num) == 0)
        {
            printf("thread[%d]: %d\n", num, count++);
            sleep(1);
        }
        printf("thread %d end...\n", num);

        pthread_exit(NULL);
    }

    int getTrigThreadStat(int dev_n)
    {
        int ret = 0;

            pthread_mutex_lock(&TrigStopMutex[dev_n]);
            ret = SendTrigThreadStop[dev_n];
            pthread_mutex_unlock(&TrigStopMutex[dev_n]);

        return ret;
    }

    void setTrigThreadStat(int dev_n, int val)
    {
            pthread_mutex_lock(&TrigStopMutex[dev_n]);
            SendTrigThreadStop[dev_n] = val;
            pthread_mutex_unlock(&TrigStopMutex[dev_n]);
    }



    int main(int argc, char *argv[])
    {
        int i;
        int thread_n = 3;
        // start thread
        if (pthread_create(&thread_SendTrigMsg[thread_n], NULL, func, (void*)thread_n)) 
        {
            printf("pthread_create %d error\n", thread_n);
            return -1;
        }

        sleep(5);


        setTrigThreadStat(thread_n, 1);
        pthread_join(thread_SendTrigMsg[thread_n], NULL);
        printf("main canceled thread %d\n\n", thread_n);
        setTrigThreadStat(thread_n, 0);

        thread_n = 1;
        if (pthread_create(&thread_SendTrigMsg[thread_n], NULL, func, (void*)thread_n))
        {
            printf("pthread_create %d error\n", thread_n);
            return -1;
        }

        sleep(8);

        setTrigThreadStat(thread_n, 1);
        pthread_join(thread_SendTrigMsg[thread_n], NULL);
        printf("main canceled thread %d\n\n", thread_n);
        setTrigThreadStat(thread_n, 0);

        return 0;
    }
